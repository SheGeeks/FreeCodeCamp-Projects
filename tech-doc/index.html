Box Model

In CSS, the box model is a way to visualize the spacing that wraps around every HTML element, aka any content. This spacing consists of:

  • margins
  • borders
  • padding

Let's dig into each of these properties to see how they work.

Padding

                    padding-top: 1rem;
                    padding-right: 1rem;
                    padding-bottom: 1rem;
                    padding-left: 1rem;

padding

Padding refers to the immediate space inside of an element's borders. You can define the padding for each side of an element by length, percentage or through inheritance.

Since the values are the same for all sides, this code can be refactored as: padding: 1rem;

You can also set the value for opposite sides of an element (top/bottom, left/right) likeso: padding: 1rem 1rem;

Each of these code snippets are valid ways to manipulate padding values. In this example, they all have the same value: 1rem of spacing around an element. The first option gives you more control, while the others can result in cleaner code.

Borders

                        p {
                         border: 5px solid red;
                        }

border

Borders wrap around an element and include in padding the element has set. Borders can be defined with a style, width, and color. Values for all three properties can be set in one line of code using the border shorthand property. Alternatively, each property can be explicity defined.

border-width

The border-width property defines how thick the border looks around an element. It can be defined in px, pt, cm, em and more. In addition, there are three pre-defined values to choose from: thin, medium, and thick. Just like padding, the border-width can be adjusted for all sides.

                        p {
                         border-top-style: dotted;
                         border-right-style: solid;
                         border-bottom-style: dotted;
                         border-left-style: solid;
                        }

border-style

Just like padding, borders apply to all four sides of an element with the same or different values applied to each side.

One value can be used to apply to all sides. border-style: dotted; and different values can set for each side. This code applies different values to all four sides: border-style: dotted solid double dashed;.

Multiple values are applied starting from the top, then to the right, bottom, and left. When using only three values the second value is applied to the right and left border. If there are only two values, the first value is applied to the top and bottom borders. This logic also applies to the border width and border-color.

border-color

The border color is self-explanatory and can be set using the property border-color. Color values can be set in RGB, HSL, and HEX.

border-radius

If you look at the corners of the white container holding this text, you'll notice they're rounded rather than straight. Creating rounded corners on any element can be done using the border-radius property. One cool thing about this property is that it can be set on any element without a border. This is great for rounding the corner of image or a content container like this one.

Margins

                    p {
                     margin-top: 3rem;
                     margin-right: 3rem;
                     margin-bottom: 3rem;
                     margin-left: rem;
                    }

margin

Margins are used to create space around elements and outside of their borders. Similar to padding and borders, the same value can be set with margin: 1rem; or different values can be set withmargin: 1rem 2rem 3rem 4rem.

Margin Collapse

Occassionaly, top and bottom margin elements can collapse into a single margin. Whichever margin has a bigger value will become the defacto margin value. Fortunately, only vertical margins collapse (top/bottom). That's because collapsing margins are supposed to make it easy to set vertical margins for sets of content, which you can learn more about in this article.
